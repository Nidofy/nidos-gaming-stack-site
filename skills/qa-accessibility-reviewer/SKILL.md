# qa-accessibility-reviewer

## name

qa-accessibility-reviewer

## description

检查站点页面的质量、移动端体验、可访问性、SEO、性能和下载流程风险的 skill。

## 何时触发

- 用户要求 review、QA、上线前检查。
- 页面、下载页、开发日志或微信落地页实现后。
- 发布前需要确认移动端、SEO、无障碍或链接状态。

## 工作步骤

1. 读取 `AGENTS.md` 和相关 spec。
2. 本地构建或运行项目，若项目尚未初始化则记录无法执行。
3. 检查页面结构、标题层级、alt、按钮文本、键盘可达性、颜色对比、移动端溢出。
4. 检查 SEO、hreflang、OG、canonical 和结构化数据。
5. 检查下载按钮是否引用 release 数据。
6. 输出按严重度排序的问题清单和建议。

## 输入材料

- 已实现页面或 PR diff。
- 本地预览 URL。
- 相关内容和 release 数据。

## 输出文件

- QA 报告。
- 必要的小修复补丁。
- 未执行检查的原因记录。

## 验收标准

- 严重问题优先列出。
- 每个问题有文件位置、影响和建议。
- 未执行的测试明确说明原因。
- 不把主观审美意见伪装成阻塞问题。

## 禁止事项

- 不在未授权情况下大规模重构。
- 不忽略移动端。
- 不只给笼统评价。
- 不绕过 release 数据检查下载链接。
