# landing-page-builder

## name

landing-page-builder

## description

为“Nido的游戏小栈”构建或修改首页、游戏详情页、专题页和产品叙事页面的 skill，强调长期作品集结构、内容驱动和移动端体验。

## 何时触发

- 用户要求实现首页、游戏详情页、产品展示页、滚动叙事页面。
- 用户要求把某个游戏接入站点展示。
- 用户要求调整 Apple 式极简产品展示体验。

## 工作步骤

1. 读取 `AGENTS.md`、`spec/PRODUCT_SPEC.md`、`spec/DESIGN_SYSTEM.md`、`spec/ROUTING_SPEC.md` 和相关内容模型。
2. 确认页面是否应由已有内容数据生成。
3. 设计页面 section：hero、作品亮点、媒体、平台、下载 CTA、开发日志入口、相关作品。
4. 实现或修改 Astro 组件，保持数据和表现分离。
5. 检查移动端布局、图片 alt、CTA、语言切换和 SEO。
6. 记录未解决内容或素材 TODO。

## 输入材料

- 游戏内容文件或草稿。
- 视觉素材、截图、视频或占位素材说明。
- 路由和语言要求。
- 相关 release 数据。

## 输出文件

- Astro 页面或布局组件。
- 共享 UI 组件。
- 内容集合条目。
- 必要的样式 token 或 Tailwind 配置修改。

## 验收标准

- 页面不硬编码单一游戏。
- 中英文结构一致。
- 移动端首屏可读且 CTA 明确。
- 媒体资源有 alt 和授权说明。
- 没有使用未经授权的商标或官方赛事素材。

## 禁止事项

- 不复制 Apple 页面结构、素材、字体或文案。
- 不把 TI 模拟器写死为唯一作品。
- 不在组件中散落下载链接。
- 不创建无法被未来游戏复用的一次性页面系统。
