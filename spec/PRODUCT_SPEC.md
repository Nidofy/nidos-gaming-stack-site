# Product Spec

## 目标

建立一个长期维护的个人独立游戏发布站，中文名定稿为“Nido的游戏小栈”，英文名定稿为“Nido's Gaming Stack”。站点当前重点展示 TI 模拟器，未来支持继续加入小游戏、模拟器、实验性游戏项目、下载页、开发日志和微信公众号推广落地页。

核心目标：

- 作为中美都能访问的官方发布入口。
- 给玩家提供清晰的项目介绍、截图/视频、下载和版本信息。
- 给媒体、社区和英文用户提供结构化项目说明。
- 给微信公众号读者提供中文传播后的落地和转化路径。

## 约束

- 不做单一游戏硬编码，所有游戏、release、日志和媒体信息应由内容或数据驱动。
- 不复制 Apple 的页面结构、素材、图标、字体或文案，只借鉴其产品叙事方法。
- 不使用未经授权的真实赛事、战队、选手、Dota、Valve、The International 等商标和视觉素材作为核心资产。
- 首阶段不要求可运行网页，优先建立规范。
- 中英文内容结构保持一致，表达可以因受众差异调整。

## 推荐实现方式

- 使用 Astro + TypeScript + Tailwind CSS + MDX。
- 使用 Astro Content Collections 管理 games、posts、wechat-campaigns、releases 或等价结构。
- 页面模板按内容类型划分：主页、游戏详情、下载页、开发日志、微信公众号落地页。
- 首页展示重点作品和作品列表，不绑定单一游戏。
- 每个游戏定义统一字段：slug、title、locale、status、summary、hero、gallery、platforms、downloads、seo、legalNotes。

## 后续 Codex 修改代码时必须遵守的检查项

- 是否仍能在不改模板的情况下新增第二个游戏。
- 是否所有用户可见的游戏信息都来自内容模型或数据源。
- 是否避免了受保护商标和未经授权视觉资产。
- 是否明确区分中文传播文案和英文国际介绍。
- 是否维护了移动端、性能、SEO、可访问性和下载一致性。
