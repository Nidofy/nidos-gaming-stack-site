# Routing Spec

## 目标

定义长期可扩展的路由结构，支持中英文、多游戏、下载、开发日志、微信公众号落地页和未来专题页面。

## 约束

- 路由不能以单一游戏为根结构。
- 中英文 URL 结构应一致，语言前缀清晰。
- 下载页必须可被独立访问，也应能从游戏详情页进入。
- 微信公众号落地页是中文传播入口，不应替代正式详情页。

## 推荐实现方式

建议路由：

```text
/                         -> 默认语言首页或语言选择策略
/zh/                      -> 中文首页
/en/                      -> 英文首页
/zh/games/                -> 中文游戏列表
/en/games/                -> 英文游戏列表
/zh/games/[slug]/         -> 中文游戏详情
/en/games/[slug]/         -> 英文游戏详情
/zh/downloads/            -> 中文下载中心
/en/downloads/            -> 英文下载中心
/zh/games/[slug]/download/ -> 单游戏中文下载页
/en/games/[slug]/download/ -> 单游戏英文下载页
/zh/devlog/               -> 中文开发日志列表
/en/devlog/               -> 英文开发日志列表
/zh/devlog/[slug]/        -> 中文开发日志详情
/en/devlog/[slug]/        -> 英文开发日志详情
/zh/wechat/[campaign]/    -> 微信公众号落地页
```

重定向策略：

- `/` 可根据浏览器语言或站点策略跳转到 `/zh/` 或展示双语入口。
- 语言切换应保持同一内容 slug。
- 缺失翻译时应有明确 fallback 策略，不静默展示错误语言。

## 后续 Codex 修改代码时必须遵守的检查项

- 新增页面是否遵守语言前缀和 slug 规则。
- 是否能为同一游戏生成中英文对应页面。
- 是否有 canonical、alternate hreflang 和 Open Graph URL。
- 微信落地页是否仍能回到正式游戏详情页和下载页。
- 是否避免新增一次性路径破坏长期结构。
