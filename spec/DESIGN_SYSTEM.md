# Design System Spec

## 目标

建立适合个人独立游戏发布站的视觉系统。整体方向为极简、清晰、产品化、强视觉层级，适合展示游戏截图、视频、下载按钮和开发日志，同时保持长期扩展能力。

## 约束

- 可借鉴 apple.com 的产品展示气质，但不得复制其具体素材、图标、字体、布局切片或文案。
- 不做只适合单个游戏的视觉系统。
- 移动端体验必须优先考虑，首屏不能依赖桌面宽屏才成立。
- 不用未经授权的真实赛事、战队、选手、Dota/Valve/The International 视觉资产。
- 设计不应被单一色系完全支配；不同游戏可以有自己的 accent color，但站点品牌层应保持稳定。

## 推荐实现方式

- 使用 Tailwind CSS，结合 CSS custom properties 定义 token。
- 基础 token 包括：背景、前景、弱文本、分割线、主按钮、次按钮、强调色、危险/警告/成功状态。
- 页面结构采用全宽 section 加内容容器，不把所有内容包进卡片。
- 首页和游戏详情页使用大标题、大媒体、滚动叙事、明确 CTA、精简技术信息。
- 下载页和日志页使用更高信息密度，但仍保持清晰层级。
- 图标优先使用开源图标库，并确认许可证；按钮内图标应服务操作，不做装饰堆叠。

## 后续 Codex 修改代码时必须遵守的检查项

- 视觉是否支持多个游戏，而不是只适配 TI 模拟器。
- 每个 section 是否有明确目的和层级。
- 移动端文字、按钮、媒体是否无重叠和溢出。
- 颜色对比度是否满足基本可读性。
- 是否使用原创或授权视觉资产。
- 是否避免过度卡片化和一次性装饰。
