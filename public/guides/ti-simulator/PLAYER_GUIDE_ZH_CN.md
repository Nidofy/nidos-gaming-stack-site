# TI15 Simulator 中文游玩指引

本文档面向收到 Windows 预览包的试玩玩家，目标是帮助你从解压、启动、开始旅程，到完成一轮 BO3 并导出反馈。

## 试玩前须知

- 这是一个非官方、虚构数据的赛事 RPG/策略模拟器。
- 队伍、选手、赛事情节均为 mock/虚构内容，不代表真实 TI15 参赛名单。
- 游戏不会模拟实时 Dota 战斗。它使用抽象统计模型，根据阵容、英雄、出装路线、策略、地图决策、士气和随机种子生成比赛结果。
- 英雄名称和部分常用术语会接近 Dota 语境，但项目不包含官方队标、官方图片或 Valve/Dota 品牌素材。
- 当前预览包未进行代码签名，Windows 可能出现 SmartScreen 提示。请只运行从本站下载页或开发者直接提供的包。

## 下载与解压

当前下载页提供两个镜像，文件名以下载页显示为准：

- GitHub Release：`TI15Simulator-portable-windows-x64-20260701-preview-r3.zip`
- 百度网盘：`TI15Simulator-portable-windows-x64-20260701-preview-r3.zip`

下载后先完整解压，再启动游戏。不要直接在 ZIP 预览窗口中运行。

## 启动游戏

1. 右键 ZIP，选择“全部解压”或用 7-Zip 解压到普通文件夹。
2. 不要直接在 ZIP 预览窗口中运行。
3. 打开解压后的文件夹。
4. 双击 `Start-Game.bat`。
5. 等待桌面窗口打开。

如果 Windows SmartScreen 提示：

1. 确认 ZIP 来自本站下载页或开发者直接提供的镜像。
2. 点击“更多信息”。
3. 如果你信任此预览包来源，再选择“仍要运行”。

如果启动失败，请把解压目录里的 `data/logs` 文件夹发给开发者。

## 第一局应该怎么玩

第一次打开后，建议按默认中文流程走完一轮，不要急着研究所有高级细节。

推荐顺序：

1. 点击开始新旅程。
2. 选择或确认你的队伍。
3. 选择你要扮演的位置。
4. 选择第一局英雄。
5. 选择第一局简单打法。
6. 开始第一个 BO3。
7. 在检查点界面选择行动卡。
8. 打完至少一局后看赛后报告。
9. 尽量完成整个 BO3。
10. 导出试玩记录并写下主观反馈。

第一局的重点不是“最优解”，而是观察：

- 你是否知道下一步该点哪里。
- 三张行动卡是否读得懂。
- 比赛结果是否能讲出一个清楚的故事。
- 赛后报告是否能解释为什么赢或输。

## 核心概念

### 旅程

旅程是一次锦标赛 run。你会带着一支队伍从瑞士轮/淘汰赛流程中继续前进，期间会保存：

- 当前战绩。
- 待打对手。
- 已完成比赛。
- 队伍士气。
- 选手士气。
- 成就进度。
- 进行中的 BO3 或检查点游戏。

### BO3

BO3 是三局两胜。每一局可以重新选择英雄、出装路线、策略、队友建议和 ban 优先级。

当前预览版推荐使用“逐局 + 检查点”方式，而不是只点完整快速模拟。这样更容易看到地图决策、士气、事件和赛后报告。

### 检查点

检查点是单局比赛中的关键时间点，例如前期转线、中期肉山、后期高地或买活窗口。

检查点不会让你操作补刀、走位或技能。它会把局势压缩成一个战术选择：

- 当前形势。
- 对手威胁。
- 地图/肉山/高地状态。
- 三张行动卡。
- 选择后的结果反馈。

### 行动卡

行动卡通常分为稳健、均衡、冒险三类。你不需要理解所有底层数值，只需要判断这张卡像不像你想做的比赛决策。

常见行动方向包括：

- 保核心发育。
- 争夺肉山视野。
- 开雾抓人。
- 换线发育。
- 高地防守。
- 带盾推进。
- 切后排或抓核心。

选择行动后，游戏会显示：

- 队友响应。
- 有利因素。
- 不利因素。
- 击杀变化。
- 经济/经验变化。
- 下一步威胁或机会。

### 士气与性格事件

队伍和选手会有士气状态。士气不会直接变成“必胜按钮”，但会轻微影响稳定性、沟通、失误风险和关键时刻表现。

性格事件是比赛叙事调味，例如暂停嘲讽、沟通误解、过早庆祝等。它们应该增强故事感，而不是淹没你的战术判断。

### 成就

成就记录你在旅程中的关键表现，例如逆转、关键指挥、肉山决策、MVP 表现或稳定军心。成就存储在本地，不需要账号。

## 界面区域说明

### 首页/旅程入口

这里用于：

- 开始新旅程。
- 继续已有存档。
- 管理导入/导出存档。
- 切换中英文 UI。

默认建议使用中文试玩。

### 赛前简报

在下一场玩家控制的 BO3 前，赛前简报会提示：

- 对手概况。
- 可能被 ban 的英雄。
- 主要威胁。
- 关键对位。
- 士气或媒体预期。
- 风险标签。

如果你连续使用同一个英雄或套路，简报里可能会出现被研究、被针对或 ban 风险。

### BP / 阵容选择

每局前你可以选择：

- 你的英雄。
- 出装路线。
- 队伍策略。
- 队友英雄建议。
- Ban 优先级。

注意：

- 不能重复选择已被双方拿到的英雄。
- 被 ban 的英雄不能再选。
- 队友建议只是建议，AI 会根据合法性和阵容需要处理。
- 英雄可以用英文名、中文名、别名或简称搜索，例如 `SF`、`AM`、`小黑`。

### 战术地图

战术地图是抽象地图，不是精确 Dota 地图坐标。它用于帮助你快速理解：

- 哪一路有压力。
- 谁在靠近肉山。
- 哪边高地有危险。
- 我方/对手英雄大致意图。
- 推荐行动和风险收益。

绿色通常表示玩家队相关信息，红色通常表示对手相关信息。

### 赛后报告

每局结束后建议先看顶部的故事摘要，再看详细标签。

常见标签包括：

- 总览：胜负、比分、时长、MVP、主要叙事。
- 队伍数据：击杀、死亡、助攻、经济、经验、GPM、XPM。
- 选手数据：十名选手的 KDA、GPM、XPM、经济和英雄。
- 时间线 / 目标：关键装备节奏、目标事件、关键团、经济/经验峰值。
- Draft / Ban-Pick：BP 结果、ban 记录、阵容解释和警告。

如果你觉得结果不合理，最好同时记录当局英雄、策略、检查点选择和赛后报告截图。

## 建议试玩路线

### 快速体验路线

适合只想看游戏大概感觉的玩家。

1. 开新旅程。
2. 用默认队伍和默认名字。
3. 选择熟悉的位置和英雄。
4. 打完第一局检查点游戏。
5. 看赛后报告。
6. 导出试玩记录。

预计 10 到 20 分钟。

### 标准反馈路线

适合给开发者有用反馈。

1. 开新旅程。
2. 自定义队名或选手名。
3. 完整打完一个 BO3。
4. 每局尝试不同策略或英雄。
5. 至少体验一次肉山/高地/防守相关检查点。
6. 查看 BO3 总结、赛后报告和成就。
7. 导出试玩记录。
8. 写下最困惑和最有记忆点的地方。

预计 30 到 60 分钟。

### 压力测试路线

适合愿意帮忙找问题的玩家。

1. 连续多局重复同一个核心英雄。
2. 连续多局重复同一个策略。
3. 观察对手是否开始 ban、针对或出现适应提示。
4. 尝试极端打法，例如一直贪发育或一直冒险开团。
5. 完成瑞士轮到淘汰赛的一段旅程。
6. 导出存档和试玩记录。

重点反馈套路是否太强、太弱、太重复或难以理解。

## 存档、导入与导出

游戏存档保存在解压目录下的 `data` 文件夹中。不要只移动某个 EXE；如果要把试玩包和存档一起移动，复制整个解压后的文件夹。

界面里可以：

- 重命名存档。
- 导出旅程 JSON。
- 导入旅程 JSON。
- 删除测试存档。
- 导出试玩记录 Markdown。

给开发者反馈时，最有用的是：

- 导出的试玩记录 Markdown。
- 问题发生前后的截图。
- 触发问题的存档 JSON。
- 如果启动失败，提供 `data/logs`。

## 反馈时请重点记录

请优先记录这些问题：

- 第一分钟是否知道该做什么。
- 哪个按钮或文案让你困惑。
- 三张行动卡是否像真实战术选择。
- 战术地图是否看得懂。
- 肉山、高地、带盾、防守等状态是否解释清楚。
- 赛后报告是否能解释胜负。
- 某个英雄、策略或重复套路是否明显过强。
- 性格事件是否太频繁、太随机或太打断。
- 中文文案是否有乱码、英文残留或读不顺。
- 是否愿意再开一轮。

## 常见问题

### 为什么没有真实队伍和选手？

当前版本只使用虚构 mock 数据。真实队伍、选手和外部数据源会等数据边界稳定后再考虑。

### 为什么比赛不是我手动操作？

这是抽象赛事 RPG/策略模拟器，不是实时动作游戏。你的主要操作是赛前准备和关键时间点决策。

### 为什么我赢面高还是输了？

模型会计算胜率，但比赛结果仍包含随机性。高胜率代表更容易赢，不代表必胜。

### 为什么同一个 seed 结果一样？

这是为了方便复现 bug 和对比调整。相同 seed、相同选择通常会产生相同结果。

### 为什么 Windows 提示不安全？

当前预览包未签名。小范围朋友试玩请只使用本站下载页或开发者直接提供的镜像；公开分发需要正式代码签名和发行信誉。

### 可以离线玩吗？

Windows 便携预览包设计为离线可运行，不需要安装 Python、Node.js 或浏览器。解压完整 ZIP 后运行 `Start-Game.bat` 即可。
